Understand a web server (a little bit)

Introduction

Download XAMPP/WAMPP to my computer and I have a server already configured. These applications abstract all the details behind.

Copy/paste commands from the Internet to configure/optimize a NGINX server is frustrating without any server knowledge.

Let’s cover the base.

Basic knowledge

PHP

  • A scripting language server-side used to build web applications.
  • PHP code can be embedded into the HTML page.
  • When a PHP file is requested, the PHP code is executed then the webserver sends the result (HTML page) to clients.

Question : Who execute PHP code ?

  • PHP software process the PHP code. Not the webserver.
  • PHP and webserver work closely, but they are different.

Generally, when receiving a request involve the web app, the webserver will tell the application to execute. The server then receives the result from the app and sends it back to the web browser.

We have this stack :
Webserver -> Interface -> Application

Each element in the stack has many implementations :

  • For webserver, we have NGINX, Apache, IIS,…
  • For interface, we have CGI, FastCGI, SuPHP, DSO,…
  • For application, we can write in many different languages such as PHP, Perl, C/C++, Python, Java,…

So an Apache or a NGINX server doesn’t necessarily run web apps written in PHP.

In 2020, when installing a new server, a popular combination is NGINX-FastCGI-PHP.

NGINX

In addition to being a traditional HTTP server, NGINX can be used as a reverse proxy server or an email server. NGINX is known for its high performance, especially when serving static files.

FastCGI

It’s a protocol allow the web server and the web application to communicate together. FastCGI tackles the problem of per-request process creation and termination in the former CGI by using persistent processes to handle many requests.

With these elements, we can now set up a NGINX server.

Setup a NGINX server

There are a lot of tutorials about how to setup/config an NGINX server. They contain a series of commands that we can copy/paste to the terminal. Sometimes it works, sometimes we encounter an error we don’t understand why. That’s the problem. In my case, instead of displaying the page, the browser downloads the file index.php.

In my opinion, an installation tutorial should introduce the system being installed: the components, the relation between them… before showing the commands.

Run the FastCGI server

All the components are installed, we need 1 more step: configuration.

The folder /etc/nginx contains all the configuration of the NGINX server. We focus on :

  • nginx.conf: The global config of NGINX
  • fastcgi_params.conf: The config of FastCGI
  • sites-available: Each file in this folder contain the configuration of 1 site.

sites-available/yourdomainname.com

Below is a basic configuration for a site written in PHP.

server {

    # The server will run on port 80
    listen 80 default_server;
    listen [::]:80 default_server;

    # The server name
    server_name yourdomainname.com; 

    # The directory where the server looks up files to serve
    root /var/www/html;
    index index.html index.htm;

    # The log file
    access_log   logs/yourdomainname_access.log;
    error_log    logs/yourdomainname_error.log;

    location / {
        try_files $uri $uri/ =404;
    }

    # Process the PHP file
    # we pass the PHP requests to the fastcgi at  unix:/var/run/php/php7.0-fpm.sock

   location ~ \.php$ {
        try_files $uri =404;
        fastcgi_split_path_info ^(.+\.php)(/.+)$;
        fastcgi_pass unix:/var/run/php/php7.0-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
 }

Start the php-fpm and nginx process. The server can process the PHP files. The stack : NGINX -> Interface -> Application is alive ! You can improve each component separately: using Gzip, pagespeed module in NGINX, use cache in FastCGI server or write better PHP code…

Conclusion

Understand the architecture of what is going to be installed is important. If not, copy/paste the code from the Internet can give an unexpected result simply because it doesn’t fulfill your needs.
